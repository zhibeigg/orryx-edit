# Design Context

## Users

- Minecraft 服务器服主与插件开发者。
- 主要在桌面端进行长时间、高频配置编辑，并涉及多人协作与交接。

## Product character

- 克制、专业、高信息密度。
- 工业化、实用主义，而非营销型视觉。
- 暗色为主，核心交互满足 WCAG AA。
- Connect、Admin、Portal 完整响应式；复杂编辑器桌面优先，并在窄屏明确降级。

## Brand palette

- 品牌基准来自像素火焰机械鱼：煤黑、炭黑、深酒红、铁锈红、朱红、熔岩橙与暖白。
- 60% 使用带红褐色倾向的煤黑表面，30% 使用酒红/铁锈红建立结构与边界，10% 使用朱红/熔岩橙强调主操作和焦点。
- 暖白与奶油白承担正文、高对比数字和成功确认；门户、账户与管理品牌层不使用青蓝、紫蓝或荧光绿色。

## Editor workbench

- 高密度 Kether/Scratch 编辑区采用 VS Code Dark+ 工作台语言：中性深灰表面、资源管理器式层级、蓝色焦点与克制的语法色。
- 分类名称面向中文用户展示，Registry 原始分类 ID 只作为技术标识和搜索兼容项保留。
- 编辑器分类色限制为蓝、青绿、浅黄、珊瑚四组，仅用于小型标记、端口和语义提示，不以整行彩虹文字制造噪音。

## Visual rules

- 使用实色分区、1px 边框、2–4px 圆角、清晰状态色和可见焦点。
- 禁止常见 AI 紫蓝渐变、毛玻璃、发光阴影、渐变文字和彩色侧边粗条。
- 优先稳定的信息层级、可扫描的数据布局和明确的操作后果说明。
