# Orryx Editor 设计上下文

## Design Context

### Users
- **主要用户**：Minecraft 服务器插件开发者与服主（Orryx 插件使用者），中文界面为主。他们在桌面浏览器中长时间编写技能脚本（Kether）、可视化 Flow 编排、任务/物品/NPC 等 YAML 配置。
- **次要用户**：站点管理员在浏览器中管理商业 License、同步 Kether Docs、处理提供商与更新任务。
- **使用场景**：高强度、长时间的创作型工作（编辑器），以及低频但要求可靠的管理操作（后台/门户）。编辑器在窄视口下有门禁提示，管理端需完整响应式。

### Brand Personality
- 三个词：**工业感、密集高效、专业可靠**。
- 调性：继承 VSCode Dark+ 的工具传统——信息密度优先于装饰，锐利边角（2-3px 圆角），无渐变、无玻璃拟态、无装饰性阴影。
- 情感目标：让用户感到这是一个"为干活而生"的专业工作台，而不是营销页面。

### Aesthetic Direction
- **深色主题为主**（开发者长时间使用、暗光环境）。不做浅色模式的表面功夫。
- **双区色板**：编辑器区使用 VSCode Dark+ 蓝（#007acc）；管理/门户区使用暖铜-琥珀强调（oklch ~70% 0.215 42），形成区域识别。
- **中性色向品牌色相倾斜**，禁止纯黑 #000 / 纯白 #fff。
- 排版：UI 使用固定 rem 比例（非流式），代码使用等宽字体；中文优先的系统字体栈可接受，但需保证字号层级对比 ≥1.25。
- 参考：VSCode 编辑器密度与交互习惯；JetBrains IDE 的工具窗口组织。
- 反面参考：营销式英雄区、渐变文字、毛玻璃卡片、弹跳动画——一律禁止。

### Design Principles
1. **密度优先**：编辑器与管理界面追求信息密度与操作效率；间距服务于分组与层级，而非"呼吸感"本身。
2. **键盘可达**：所有交互可键盘完成；焦点环统一使用 `--focus-ring`，WCAG AA 对比度（4.5:1）。
3. **颜色只用令牌**：禁止在组件中硬编码 hex；统一收敛到 design-tokens / industrial 令牌体系。
4. **动效克制且可降级**：仅 transform/opacity 动画，尊重 prefers-reduced-motion；动画用于状态反馈，不用于装饰。
5. **适配而非截断**：管理端/门户全响应式；编辑器在移动端给出明确门禁说明而非破碎布局。

### Technical Constraints
- React 19 + TypeScript + Vite 8 + Tailwind CSS 4（@theme 令牌）+ Radix UI + @xyflow/react + Monaco。
- 已具备：skip-link、全局 :focus-visible 环、路由级懒加载、bundle 预算检查脚本。
- 目标浏览器：现代桌面 Chrome/Edge/Firefox；管理端支持移动浏览器。
