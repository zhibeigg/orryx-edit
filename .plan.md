# Skill-Station 交叉引用分析计划

## 目标

在编辑器中新增"引用"标签页，自动扫描所有已加载文件，分析 skill 和 station 之间的交叉引用关系，让用户一目了然地看到：
- 某个 flag 在哪些文件中被设置/检查/消耗
- 某个 cooldown 在哪些文件中被操作
- 某个变量（`orryx skill var 技能名 *变量名`）在哪些 station 中被读取
- 某个技能的 cooldown 在哪些 station 中被引用

## 实现方案

### 1. 新增 `web/src/lib/cross-ref-analyzer.ts` — 交叉引用分析引擎

扫描所有文件内容，用正则提取引用关系：

```typescript
interface CrossRef {
  type: "flag" | "cooldown" | "variable" | "cast" | "skill-var" | "skill-cooldown"
  name: string           // flag 名、变量名、技能名
  action: "set" | "check" | "remove" | "consume" | "read"
  file: string           // 文件路径
  line?: number          // 行号
  snippet: string        // 匹配到的代码片段
}
```

正则模式：
- `flag\s+(\S+)\s+to` → flag 设置
- `flag\s+(\S+)\s+remove` → flag 移除
- `if\s+flag\s+(\S+)` / `flag\s+(\S+)\s+[><=!]` → flag 检查
- `cooldown\s+set\s+(.+)` → cooldown 设置
- `orryx\s+skill\s+var\s+(\S+)\s+\*(\S+)` → 跨文件读取技能变量
- `orryx\s+skill\s+cooldown\s+(\S+)` → 跨文件读取技能 CD
- `cast\s+(\S+)` → 技能调用
- `buff\s+send\s+(\S+)` / `buff\s+check\s+(\S+)` → buff 引用

### 2. 新增 `web/src/components/editor/CrossRefPanel.tsx` — 引用面板组件

在 SkillEditor 中新增"引用"标签页，显示当前文件中所有标识符的交叉引用：

- 按引用类型分组（Flag / Cooldown / 变量 / Cast）
- 每个引用项显示：引用名、操作类型（设置/检查/消耗）、引用文件列表
- 点击文件名可跳转打开该文件
